English

This tutorial is for correctly compiling the project, then assembling the artifact and being able to generate the editor's .jar file.

UPDATE 16-4-2026: Since the project uses Gradle, just follow step 1: clone the repository on GitHub, open the cloned folder with Visual Studio Code (it should be Desperados_Mission_Editor), and it will detect the Gradle project. In the task list, use `build` to generate the .jar file and `run` to run the .jar file and test the app. If it asks you to install Java extensions, Gradle, etc., you must do so for it to detect Gradle and the Java project (wait, it takes some time. Restart Visual Studio Code if prompted).

1- Install "OpenJDK8U-jdk_x64_win_8u482b08.exe", this is java dk (development kit) v8, for old projects like this.

2- Install and open the IntelliJ IDEA 2026 program (it may work with another program, I use this one).

3. Drag or open the "DesperadosMissionEditor" folder (the folder containing this tutorial) into IntelliJ.

4. Verify that the "scr" folder is marked as "sources root" (blue). Hover your mouse over it, and it will tell you. If nothing appears, we need to specify it. Right-click, select "Mark Directory As," and then "Sources root." If the option doesn't appear, try a different directory and see if it appears again.

5. Click on the IntelliJ main menu (three horizontal lines in the upper left corner of the screen) and then "Project Structure."

6. In the Project tab:

      Name: Anything you want, but try not to leave any spaces.

      SDK: temurin-1.8 (Eclipse Temurin 1.8.0_482). If it doesn't appear, download the JDK, version 1.8, vendor temurin-1.8, and download it.

      Language level: 8 - Lambdas, type annotations, etc.

   In the Modules tab:

      Module SDK: temurin-1.8. If it doesn't appear, do the same as you did in the previous tab.

      In the container, it should show the module name and below it the same libraries that are in the project's "lib" folder. If they are not all there, you must add them one by one using the + button, Library, Java.

   In the Artifacts tab:

      Click the + button, JAR, From modules with dependencies. In Main Class, choose the MainGUI class, which is the one that contains the app's runtime, OK, Apply, OK.

7- Click on the IntelliJ main menu, Build, Build Project. In the notifications, check that it was done correctly.

   Now, click Build, Build Artifacts, select the JAR file that appears, and click Build. The .jar file will be created in the path: classes, artifacts, jarname, .jar. This is the default if you haven't changed the directory in project structure, project.

8- To properly open the generated .jar file, we need to go to the generated file on the disk and open it with WinZip, WinRAR, 7-Zip, etc. Go to the META-INF folder and delete all files with the SF, RSA, and DSA extensions. This is to remove the signatures that protect the file.







Note: The JAR file "org.eclipse.swt.win32.win32.x86_64_3.105.0.v20160603-0902" is included in case there are any incompatibilities when compiling the project. You may not need to use it.
